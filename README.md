# بازی دوز ایستاده (Connect Four)

سلام! این یه پروژه بازی دوز ایستاده هست که با زبان C نوشتم. خیلی راحته و تو ترمینال اجرا میشه.

## این بازی چیه؟

اگه یادتون باشه بچه که بودیم این بازی رو با دوستامون بازی میکردیم، اون بازی که باید 4 تا مهره رو کنار هم ردیف کنی. این همونه! ولی همینجا رو کامپیوترت میتونی بازیش کنی.

## چطور اجرا کنم؟

خب اول باید کامپایلش کنی:

```bash
gcc main.c -o connect4
```

بعدش راحت اجراش کن:

```bash
./connect4
```

## چه قابلیت هایی داره؟

خب ببین چندتا حالت داره این بازی:

### 1. بازی دو نفره (Human vs Human)
همون بازی معمولی دیگه. تو و دوستت یا هرکی که دوست داری باهاش میتونی بازی کنی. نوبتی یکی یکی مهره هاتونو میندازید تو ستونها.

### 2. بازی با فایل (Human vs File)
این یکم خاصه! یه فایل متنی میسازی، توش شماره ستون هایی که میخوای مهره هات بیفته رو مینویسی (هرکدوم تو یه خط جدا)، بعد برنامه خودش حرکت های اون فایل رو اجرا میکنه. خیلی خوبه واسه تست کردن!

### 3. بازی با کامپیوتر (Human vs Computer)
اینجا خودت با کامپیوتر بازی میکنی. دوتا سطح سختی داره:
- **آسون:** کامپیوتر یه ستون رندم انتخاب میکنه، خیلی راحت میتونی ببریش!
- **سخت:** این دیگه یکم چالشی تره! کامپیوتر یه فکرای باسواد میکنه. اگه ببینه میتونه ببره همون حرکت رو میزنه، اگه ببینه تو داری میخوای ببری جلوتو میگیره.

### 4. نگاه دوباره به بازی قبلی (Replay)
این خیلی باحاله! اگه بازیتو ذخیره کرده باشی، بعدا میتونی بیای اینجا و تمام حرکت هاتونو یکی یکی ببینی. مثل فیلم که داره دوباره پخش میشه!

## ذخیره کردن بازی

وقتی بازی تموم میشه، ازت میپرسه میخوای ذخیره کنی یا نه. اگه گفتی آره، یه اسم بهش بده (بدون فاصله، مثلا game1 یا بازی_من) و برنامه همه چیزو تو فایل database.txt میریزه.

بعدا هروقت خواستی میتونی از منوی اصلی گزینه 4 رو بزنی و اون بازی رو ببینی.

## سایز زمین بازی

وقتی بازی رو شروع میکنی، ازت میپرسه چند تا ردیف و چند تا ستون میخوای. فقط یادت باشه:
- حداقل 4 تا ردیف باید باشه (والا که نمیشه 4 تا مهره رو کنار هم چید!)
- ستونا هم باید کمتر از 12 تا باشن

## فایل حرکت ها چطوری بسازم؟

خیلی سادست. یه فایل متنی بساز، تو هر خط فقط یه عدد بنویس که شماره ستونه. مثلا:

```
3
2
4
1
```

این یعنی اول مهره رو تو ستون 3 بنداز، بعد ستون 2، بعد 4، و بعد 1. دقت کن ستونا از 0 شروع میشن!

## توضیحات کد

کد خودش خیلی تمیز و منظم نوشته شده. اگه میخوای یاد بگیری یا یکم دستکاریش کنی، اینا تو کد پیاده شدن:

- **ساختار داده ها:** همه چیز با struct نگهداری میشه. وضعیت بازی، اطلاعات بازیکنا، همه چی
- **مدیریت زمین بازی:** تابع هایی که زمین رو میسازن، چاپ میکنن، مهره ها رو میندازن تو ستونا و چک میکنن کی برده
- **ذخیره و بازپخش:** با کار با فایل پیاده شده. همه حرکت ها تو یه فایل ذخیره میشن
- **هوش مصنوعی:** تو حالت سخت، کامپیوتر با یه الگوریتم ساده ولی موثر بازی میکنه
